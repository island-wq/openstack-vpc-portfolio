# 문제와 목표

## 문제 정의

- 제품 기능 설명만으로 실제 운영 적합성 판단 곤란
- 고객 네트워크·보안·스토리지 정책의 사전 확정 부재
- 워크로드 이관과 플랫폼 기능 검증의 혼재
- 기능 수행 여부와 합격 판정 기준의 연결 부재
- 짧은 PoC 기간 내 전 항목 검증의 현실성 부족

## 목표

- 요구사항별 시험 목적·절차·증적·판정 기준 정의
- 이관형과 플랫폼 검증형 PoC의 분리
- 물리 연결부터 논리 자원까지 추적 가능한 아키텍처 수립
- 정상 기능과 장애·복구 기능의 균형 검증
- 미검증 항목과 운영 전제조건의 명시

## 성공 기준

- 필수 이관 대상의 데이터·애플리케이션 정합성 확인
- 인스턴스·네트워크·볼륨 생명주기 기능 확인
- Live/Cold Migration과 Host Evacuation 절차 확인
- Snapshot·Backup·Restore 경로 확인
- 보안그룹과 Project 격리 정책 확인
- 고객 인수 기준에 대응 가능한 결과 증적 확보
