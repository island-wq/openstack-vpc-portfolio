# 성과와 로드맵

## 설계 성과

- 이관형과 플랫폼 검증형 PoC 모델 정립
- 물리·논리·서비스 흐름의 연결 구조 수립
- 기능·운영·장애·백업 시험 시나리오 정의
- Project 격리·DVR·Cinder 기반 핵심 검증 범위 정리
- 필수·정량·정책 연계 항목의 분류
- 미결정 조건과 외부 의존사항의 조기 식별

## 확인 필요 사항

- 첨부 자료의 성격이 계획·설계 중심
- 전체 Test Case별 실제 실행 결과 부재
- 성공률·성능·복구시간의 최종 측정값 부재
- 운영 승인·보안 적합성의 최종 판정 부재
- 포트폴리오 내 결과 표현의 과장 방지 필요

## 단계별 로드맵

```mermaid
flowchart LR
  p1["Phase 1<br/>요구사항·범위 확정"]
  p2["Phase 2<br/>시험 자동화·증적 수집"]
  p3["Phase 3<br/>장애·성능 기준 정량화"]
  p4["Phase 4<br/>운영 전환 판단"]

  p1 --> p2 --> p3 --> p4
```

### Phase 1

- 고객망·보안·스토리지 정책 확정
- 필수 항목과 제외 항목 합의
- 합격 기준과 책임 구분 확정

### Phase 2

- 반복 시험의 자동화
- API·CLI 기반 결과 수집
- 증적 파일명·보관 위치 표준화

### Phase 3

- Migration 중단시간 측정
- Evacuation·Backup 복구시간 측정
- CPU·메모리·IOPS·네트워크 기준선 확보

### Phase 4

- 잔여 위험과 예외 조건 검토
- 상용 구성과 PoC 축소 구성의 차이 분석
- 운영 전환·추가 검증·도입 보류 판정
