# 검증 범위와 판정

## 검증 매트릭스

| 영역 | 대표 항목 | 증적 | 판정 |
|---|---|---|---|
| 이관 | VM·DB·Image 변환 | 정합성 비교·부팅 결과 | 필수 |
| 컴퓨트 | 생성·Resize·Migration | 상태·로그·Host 변경 | 필수 |
| 네트워크 | 격리·라우팅·Security Group | 통신 성공·차단 결과 | 필수 |
| 스토리지 | Attach·Extend·Snapshot·Backup | Volume 상태·복원 결과 | 필수 |
| 장애 | Host Evacuation·경로 장애 | 복구 시간·서비스 상태 | 조건부 |
| 운영 | 모니터링·로그·알림 | Dashboard·Event 기록 | 조건부 |
| 보안 | 취약점·정책·승인 | 검사 결과·통제 기록 | 정책 연계 |

## 판정 기준

- 성공: 목표 결과 충족과 재현 가능한 증적 확보
- 조건부 성공: 기능 확인과 운영 제약 존재
- 실패: 목표 결과 미충족 또는 데이터 정합성 훼손
- 미검증: 환경·일정·권한 부재로 실행 불가

## 결과 기록 원칙

- 시험 ID와 요구사항 ID 연결
- 사전조건과 입력값 기록
- 실제 수행 절차 기록
- 결과 화면·로그·상태값 확보
- 장애 발생 시 원인·우회·재시험 이력 기록

## 공개 범위

- 고객명·시스템명 비공개
- 실제 IP·VLAN·장비 식별자 비공개
- 계정·보안정책·취약점 상세 비공개
- 설계 판단·시험 구조·교훈 중심 공개
