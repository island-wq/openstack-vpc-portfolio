# 문제와 목표

## 배경

### 소규모 고객에게 과도한 초기 구성

기존 표준 상품은 Controller, Compute, Storage를 역할별로 분리해 안정적이지만, 소규모 고객에게는 랙 공간·장비 수·초기 비용의 부담

### 소형 제품의 확장성 한계

상용 워크로드에 필요한 고가용성, 분산 스토리지, 노드 단위 확장을 동시에 제공하기 어려웠습니다.

### 설치 방식의 편차

장비 별 수동 설치에 의존하면 구축 기간과 작업 품질이 담당자 경험에 따른 편차 발생 우려
상품으로 반복 공급하기 위한  드웨어 기준과 운영체제 배포 절차 표준화 진행

## 프로젝트 목표

### 1. 상품 라인업의 중간 구간 정의

- 소형 가상화 제품보다 높은 가용성과 확장성 제공
- 표준 랙 상품보다 적은 장비와 공간으로 시작
- 노드 증설을 통해 컴퓨트와 스토리지를 함께 확장

### 2. 역할 통합과 장애 허용의 균형

- 3개 HCI 노드에 Controller, Compute, Ceph 역할 배치
- 관리·배포 기능은 별도 노드로 분리
- LACP와 OpenStack HA를 통해 단일 장애점 축소

### 3. 반복 가능한 구축 절차 정의

- IPMI 기반 하드웨어 원격 관리
- DHCP, TFTP, Repository를 이용한 PXE 배포
- 논리 네트워크와 노드 역할을 선언적으로 관리

## 성공 기준

| 관점 | 기준 |
| --- | --- |

| 아키텍처 | 3노드에서 제어·컴퓨트·스토리지 고가용성 구성 |
| 네트워크 | 관리, API, Provider, Tenant, Storage 트래픽 분리 |
